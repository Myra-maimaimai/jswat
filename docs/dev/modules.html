Writing New Modules

Writing new modules for JSwat is as easy as writing new modules for the NetBeans Platform. You can use the NetBeans IDE 5.0 or later to conveniently set up a module project. If you do so, be sure to add the copy of the NetBeans Platform used for JSwat to the Platform Manager in the IDE. Alternatively, you may just want to start by copying files from one of the JSwat modules, which are developed without the aid of the API support features of the NetBeans IDE.

See the instructions on compiling JSwat to get your build environment set up correctly before proceeding.

File Layout

Please use the following directory structure when creating new modules for JSwat. This is basically the standard layout for any NetBeans module.

  [module-name]/
     build.xml
     manifest.mf
     src/
        com/bluemarsh/jswat/**/*.java
        com/bluemarsh/jswat/**/Bundle.properties
        com/bluemarsh/jswat/**/mf-layer.xml
        com/bluemarsh/jswat/**/package.html
     test/
        unit/
           src/
              com/bluemarsh/jswat/**/*Test.java

Dependencies

Add the following line to your module manifest.mf file; this will prevent automatic dependencies on NetBeans modules that you probably do not need. In fact, if you do not have this dependency, then your module will be disabled automatically by the Platform.

OpenIDE-Module-IDE-Dependencies: IDE/1 > 6.0
