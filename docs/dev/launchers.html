OS-specific Launchers

Microsoft Windows

Most of the users of JSwat are running some variant of Microsoft Windows, most likely 2000 or XP. The launcher for this platform is contained in the launcher/windows directory. This includes the C++ source file for the launcher, the makefile to compile it, an icon file, a resource descriptor file, and the compiled executables.

If and when it is necessary to recompile the Windows launchers, it must be done on a Windows system running Cygwin, the Linux emulator for Windows. Not only must the machine have the base Cygwin components, it must also have the development tools, which includes make, gcc, and MinGW.

To recompile the Windows launchers, start the Cygwin bash shell, change to the launcher/windows directory, and invoke make. This will completely rebuild the launchers from source.

Note that the jswat.exe executable does not display a command prompt, while the jswatw.exe does. This is by design, and useful for debugging.

The launcher code is simply a modification of the NetBeans launcher. The changes include the default clusters list, the branding token, and the removal of a class import. In addition, the Makefile has the patch from issue 53136 in the NetBeans IssueZilla applied to it, with some extra cleanup and the addition of the -DWINMAIN definition.

Unix-like systems

For most, if not all, of the Unix-like systems, the launcher of choice is the Bourne shell script named jswat in the directory launcher/unix. Modifying this file is trivial and no recompilation is necessary.

The launcher code is simply a modification of the NetBeans launcher. The changes include the default clusters list, the branding token, and the removal of a class import.
