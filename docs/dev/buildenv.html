Setting up the Build Environment

Installing Required Software

Java Development Kit

JSwat is written in the Java language, version 6.0. Visit the Java SE page and download the release appropriate for your system. It is possible, though not recommended, to use compatible software from other vendors. In this case, make sure that software includes the Java Platform Debugger Architecture (JPDA) library.

Apache Ant

Building JSwat requires the use of Apache Ant, a build tool written in Java. In general you should use the latest stable release, but you may find that a slightly older release will work as well. Chances are you will have to set the JAVA_HOME environment variable to point to the JDK installation, and possibly also ANT_HOME, which points to the Ant installation directory.

Alternatively, you can use the NetBeans IDE and rely on its built-in Ant, rather than installing a separate version.

Project Settings

In order for the Java projects (core, cmd, and cli) to compile, you will need to tell Ant where to find the Java Debug Interface (JDI) classes. Normally these are located in a file called tools.jar in the JDK lib directory. That jar will contain classes in a package called com.sun.jdi and those are needed by JSwat. On the Mac, these classes are in the main classes.jar archive, so no extra setup is needed, just build the projects.

For all other systems, you will need to set the classpath for the three Java projects to include the jar for the JDI classes. See below for the details. Note that if your JDK does not have a tools.jar, or that archive does not contain the JDI classes, then you will need to locate those classes and specify that file and path in place of all the references to tools.jar below.

Core Project

Create the file core/nbproject/private/private.properties with the following contents, creating any missing parent directories:

javac.classpath=\
    lib/org-openide-util.jar:\
    lib/com-bluemarsh-jswat-parser.jar:\
    ${java.home}/../lib/tools.jar

Commands Project

Create the file cmd/nbproject/private/private.properties with the following contents, creating any missing parent directories:

javac.classpath=\
    ../core/dist/com-bluemarsh-jswat-core.jar:\
    ../core/lib/org-openide-util.jar:\
    ${java.home}/../lib/tools.jar

Console Project

Create the file cli/nbproject/private/private.properties with the following contents, creating any missing parent directories:

javac.classpath=\
    ../core/dist/com-bluemarsh-jswat-core.jar:\
    ../cmd/dist/com-bluemarsh-jswat-cmd.jar:\
    ../core/lib/org-openide-util.jar:\
    ../core/lib/com-bluemarsh-jswat-parser.jar:\
    ../bcel/release/modules/ext/bcel-5.2.jar:\
    lib/commons-cli-1.2.jar:\
    ${java.home}/../lib/tools.jar
