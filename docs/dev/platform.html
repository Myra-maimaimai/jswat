How to Update the Platform

The page documents the process for updating the NetBeans Platform on which JSwat is based. For instance, going from NetBeans 5.5 to 6.0 was a bit of an undertaking. These are the basic steps to follow in making the transition.

Update Process

  1. Copy the new org-openide-util.jar from NetBeans platform directory to the core/lib folder of the core project. Note that this file is not used when running in NetBeans.
  2. Update cluster values ide*, platform*, jswat* (e.g. ide7 to ide8):
    • build.xml
    • nbproject/platform.properties
    • launcher/unix/jswat
    • launcher/windows/jswat.cpp
    • Installer source code
  3. Run product/mkmodules.sh to recreate disabled modules list
  4. Recompile the launcher for Windows.
  5. Look for API changes in the NetBeans modules.
  6. Clean and build to check for problems.
    • Look for exceptions, warnings, or other error messages in the messages.log file.
    • Check that the version number is replaced during the build (e.g. should not see @version@ in the About dialog).
    • Run through the release testing steps to make sure the basic debugging functionality is working.
