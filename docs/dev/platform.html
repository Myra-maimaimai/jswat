How to Update the Platform

This page documents the process for updating the NetBeans Platform on which JSwat is based. For instance, going from NetBeans 5.5 to 6.0 was a bit of an undertaking. These are the basic steps to follow in making the transition.

Update Process

  1. Copy the new org-openide-util.jar from NetBeans platform directory to the core/lib folder of the core project. Note that this file is not used when running in NetBeans.
  2. Update the module dependencies:
    1. Open the module suite in NetBeans and disable all of the modules in the Libraries section of the project properties.
    2. Select the Platform cluster since we need all of the modules in the platform.
    3. Open the IDE cluster and enable the Lexer to NetBeans Bridge module (for syntax highlighting to work).
    4. Open the IDE cluster and enable the Plain Editor module (for plain text editing functionality).
    5. Open the IDE cluster and enable the User Utilities module (for Open File item in File menu).
    6. Open the Java cluster and enable the Java Editor module (for basic Java editor functionality).
    7. Open the Java cluster and enable the Java - Navigation module (for Java support in Navigator view).
    8. Click the Resolve button to automatically correct the module dependencies.
    9. Finally, be sure to Clean and Build for the module changes to take effect.
  3. Look for API changes in the NetBeans modules.
  4. Clean and build to check for problems. Check for any deprecated API warnings.
  5. Run the debugger and look for exceptions, warnings, or other error messages in the Output window of the IDE.

Test Procedure

It seems that when upgrading to a new release of NetBeans that some features stop working correctly. Ones that are often overlooked include the following.

  • node double-click
  • node right-click actions
  • Find feature in Classes view (activating, exposing matches)
  • expanded node refresh
