How to Build as an IDE Plugin

JSwat normally builds and runs as a standalone NetBeans Platform application. That is, it defines its own menu structure and default window arrangement, as well as a splash screen and about dialog. However, it is also possible to build and run JSwat as a "plugin" for the NetBeans IDE.

Note that at the moment, it is not possible to compile JSwat as a plugin to the NetBeans IDE. It had been working when the build process was completely customized. But now that it relies on the standard NetBeans module build setup, we can no longer build the plugin. At some point, need to devise a build setup for the plugin such that it can co-exist with the standalone process.

Prerequisites

To start with, ensure that your build environment is set up to compile JSwat.

Preparation

Building the plugin form of JSwat is basically a matter of modifying a property file. The file is not checked in to revision control, so modifying it does not pollute your pending change list.

build.properties

building.plugin = yes
test.user.dir = build/ideuserdir

The building.plugin property is the most important, obviously, as that controls the building of the plugin module itself. It also disables the product module, and the usual product branding.

Setting test.user.dir is optional, but highly recommended as it prevents polluting the JSwat test userdir with files and settings pertinent only to the IDE binary.

Building the Plugin

Now that the preparation is complete, simply clean and build the entire JSwat module suite, and rebuild the JSwat Plugin module. The plugin module is not a part of the module suite because it has different module dependencies than the standalone form of JSwat, so it is preferable to keep it separated logically.
