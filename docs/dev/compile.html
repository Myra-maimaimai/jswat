How to Compile, Run, and Debug JSwat

The following is an abbreviated set of steps for building JSwat from source code. If you encounter complications at any point, please refer to the detailed steps given in the following sections.

  1. JDK 1.6 must be installed to compile JSwat.
  2. NetBeans IDE 5.5 must be installed, as JSwat is based on the NetBeans Platform, with some additional IDE modules.
  3. Make the build: ant build
  4. Run the debugger: ant run

If that worked for you, then you are on your way to developing JSwat. However, if there was a problem, continue reading the following sections for detailed information.

Installing Required Software

Java Development Kit

JSwat is written in the Java language, version 1.5. Visit the Java SE page and download the release appropriate for your system. It is possible, though not recommended, to use compatible software from other vendors. In this case, make sure that software includes the Java Platform Debugger Architecture (JPDA) library.

Presently the IBM JDK 1.5 contains a flawed xalan implementation in its boot classpath, so it is not possible to run JSwat (and likewise, NetBeans) using this particular JDK. See the NetBeans User FAQ for more information.

NetBeans IDE

JSwat is based on the NetBeans Platform 5.5, and utilizes some of the modules provided in the NetBeans IDE. Thus, to build JSwat you will need to install the NetBeans IDE version 5.5. Developing JSwat within the NetBeans IDE is highly recommended, as it makes many of the mundane tasks much easier.

You may also want to download the NetBeans IDE source code and API documentation, which can be found at the URL given above.

Building JSwat

Once the JDK and NetBeans are installed, you are ready to build JSwat. Prior to building JSwat for the first time, it is necessary to start the NetBeans IDE and open the JSwat module suite project. This will create the necessary build properties files, allowing the application to be built from the command-line. Of course, once the IDE is started and the projects opened, you may as well build from within the IDE.

To build the application from the command-line, invoke the build target on the build.xml in the root of the source tree. This will compile all of the modules and produce the JSwat modules.

> ant build

Running JSwat

Invoke the run target of the top-level build.xml file. You should soon see the JSwat splash screen followed by the main window.

> ant run

Debugging JSwat

Invoke the debug target of the top-level build.xml file. The application will start in debug mode and the IDE debugger will be connected to it automatically. Note that this target only works inside the NetBeans IDE.

> ant debug
