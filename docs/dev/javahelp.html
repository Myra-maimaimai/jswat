How to Write Help Topics

The built-in help for JSwat is written using JavaHelp, which is essentially HTML pages with XML meta-data. The help for JSwat currently resides in the help module.

If you are unfamiliar with JavaHelp, be sure to read the documentation available at the JavaHelp product page.

Mapping Topic Names

The first step in creating a new help topic is to define the unique name of the topic. For example, jswat-attach-debuggee refers specifically to the action/attach-debuggee.html help page. This topic name is used throughout the JavaHelp meta-data, as well as within the JSwat source code. The mapping is defined in the jswat-map.xml file.

Table of Contents

The new help topic must be added to the table of contents file, jswat-toc.xml. You may add the same topic multiple times, each with a different descriptive text.

Help Index

Similar to the table of contents is the help index. This is defined in the file jswat-idx.xml and is fairly self-explanatory.

Writing the Topic

The help pages themselves are written in HTML. You should be able to use any HTML 4 tags, although you should avoid using frames. XHTML is also okay, accept that the JavaHelp software does not parse terminated empty elements (e.g. <br/>), so use the HTML format for empty elements (e.g. <br>).

Referring to the Topic

Making references to the help topic is done through the NetBeans help APIs. Specifically, when displaying a dialog you can associate a help topic with that dialog. Likewise, actions and top components also can have help topics. The name used in the code should be identical to the name of the help topic in the jswat-map.xml file.
